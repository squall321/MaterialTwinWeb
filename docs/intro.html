01

왜 바꾸는가 — 협의체 비용에서 내작 AI로

외부 물성 관리 플랫폼에 대한 투자·구독 비용을 절감하고, 대신 우리 데이터에 최적화된 AI 관리 프로그램을 직접 만든다. 데이터 주권과 맥락을 우리가 쥔다.

AS-IS · 외부 협의체 플랫폼

정제값만, 맥락은 사라진다

  • ✕고가의 라이선스·구독 비용 지속 지출
  • ✕정제된 대표값만 저장 — raw·맥락 유실
  • ✕측정자·장비 편차를 사후에 추적 불가
  • ✕LLM·AI 워크플로와 단절된 폐쇄 포맷
→
TO-BE · 내작 AI 물성 플랫폼

raw + 맥락까지, AI가 읽는다

  • ✓비용 절감 — 우리 인프라·우리 데이터
  • ✓정제값·raw·측정 맥락을 한 레코드로 연결
  • ✓맥락 기반 측정 편차·문제 사후 추론
  • ✓MCP로 LLM이 직접 조회·분석·카드 생성
02

AI-ready 물성 DB — 세 겹의 데이터

측정 결과를 "숫자 하나"로 남기지 않는다. 정제값·원시신호·맥락을 함께 저장해야 비로소 데이터가 검증 가능하고, AI가 추론할 수 있는 자산이 된다.

LAYER 1

정제 측정값

Refined properties

표준 절차로 도출한 대표 물성. 탄성계수·항복·인장강도·연신율, 진응력·구성방정식·완화 Prony까지.

E · Rp0.2 · UTSHollomon/Swift/Voce/JCProny E(t)
LAYER 2

Raw 측정 데이터

Raw signals

시간-하중-변위 원시 곡선 전체를 손실 없이 보존. 언제든 재해석·재계산이 가능한 검증의 근거.

force–disp 곡선Parquet 원본재파싱 가능
LAYER 3

측정 맥락 (Context)

Material metadata + rig setup

소재 모델 메타데이터와 시험 장비 셋업을 함께 기록. 누가·어떤 장비·어떤 조건으로 측정했는지.

소재 모델 메타장비 셋업측정자·환경
핵심 목표 · 맥락으로 편차를 추론한다

실제 측정 데이터의 신뢰성을 판별하는 곳이 바로 맥락이다. raw + context를 잇는 DB는 측정하는 사람의 편차와 문제점을 사후 추론(post-hoc inference)할 수 있게 한다 — 어떤 장비 셋업이, 어떤 조건이, 어떤 측정자가 산포를 만들었는지를 데이터로 되짚는다.

04

앞으로 — 측정에서 인사이트까지, 업무 loss 0

시험 장비를 연동하고, 측정을 완전 자동화한 뒤, 측정과 인사이트 피드백 사이의 업무 손실을 없애는 Agent로 진화한다.

PHASE 1
구현 완료

물성 DB + LLM MCP

실측 기반 DB, 인사이트 대시보드, LLM이 물성을 가져오는 MCP까지 동작.

PHASE 2
다음 단계

시험 장비 연동

장비의 raw·셋업·소재 메타를 자동 수집해 AI-ready DB에 적재. ABYSS부터.

PHASE 3
계획

측정 완전 자동화

장비 제어·측정·적재·물성 도출까지 사람 개입 없이 파이프라인화.

PHASE 4
계획

측정 Agent화

측정과 인사이트 피드백 사이의 업무 loss 최소화 — Agent가 편차를 감지·피드백.
